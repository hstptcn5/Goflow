# Goflow Trademark Guidelines

These guidelines explain how the Goflow name, logo, and branding may be used.

The software code in this repository is licensed under the license stated in `LICENSE`. That software license does not grant trademark rights.

---

## Allowed Uses

You may use the Goflow name to:

- Refer truthfully to the Goflow project.
- Describe compatibility, such as "plugin for Goflow" or "workflow template for Goflow".
- Link to the official Goflow repository.
- Discuss, review, benchmark, teach, or document Goflow.
- Fork the project while making clear that your fork is independent.

---

## Not Allowed Without Permission

You may not:

- Use the Goflow name or logo in a way that suggests official endorsement.
- Sell or host a competing product under the Goflow name.
- Register domains, package names, app store listings, or social handles that imply you are the official Goflow project.
- Use confusing names such as "Goflow Cloud", "Goflow Enterprise", or "Official Goflow" for an unofficial product.
- Remove or obscure attribution to the original project when presenting a fork.

---

## Forks and Modified Versions

Forks and modified versions should use a distinct name unless they are clearly presented as unofficial.

Acceptable examples:

- "Acme Workflow, based on Goflow"
- "Unofficial Goflow fork by Acme"
- "Goflow-compatible plugin"

Avoid names that make users believe the project is official.

---

## Logo Use

Do not use the official Goflow logo or visual identity for a commercial product, hosted service, or fork without permission.

You may use screenshots of the Goflow UI for documentation, reviews, and tutorials, as long as they are not misleading.

---

## Commercial Use

Commercial use of the MIT-licensed Community code is allowed by the software license. Commercial use of the Goflow brand is separate and may require permission.

For commercial licensing, hosted services, or partnership requests, see `COMMERCIAL.md`.
